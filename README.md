# 📈 Customer Sales Forecasting

## 📌 Overview

This project presents an end-to-end customer analytics and sales forecasting solution developed over four phases. It covers database design, data warehousing, exploratory data analysis, forecasting, and deployment of an interactive web application.

The project demonstrates the complete data pipeline, starting from raw customer data management to predictive analytics and user-friendly deployment.

---

## 🎯 Project Objectives

- Design and manage a customer database using SQL.
- Build a data warehouse for analytical processing.
- Perform data extraction, cleaning, and preprocessing using Python.
- Conduct exploratory data analysis (EDA).
- Develop a sales forecasting model.
- Integrate cloud-based data services.
- Deploy an interactive forecasting application using Streamlit.

---

## 📅 Project Workflow

### Week 1 – Database Management

- Designed the customer database.
- Managed customer information and interactions.
- Implemented SQL queries for data extraction and analysis.

### Week 2 – Data Warehousing

- Built a SQL Data Warehouse.
- Connected Python with SQL Server.
- Extracted and prepared customer data for analysis.

### Week 3 – Data Analysis & Forecasting

- Performed Exploratory Data Analysis (EDA).
- Built a forecasting model for future sales prediction.
- Integrated Azure services for data management.

### Week 4 – Deployment

- Developed an interactive Streamlit application.
- Deployed the forecasting model.
- Prepared the final report and project presentation.
